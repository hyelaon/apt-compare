# 수집 절차서 (브라우저 자동화 플레이북)

Claude가 사용자의 로그인된 크롬을 구동해 사이트별로 데이터를 모으는 순서.
자동 갱신 때도 이 절차를 그대로 반복한다. 각 단계 원본은 `data/raw/`에 스냅샷 저장.

## 0. 준비
- Chrome 즐겨찾기 Hyejin 폴더의 사이트 사용.
- 로그인 필요한 사이트(호갱노노 등)는 사용자가 미리 로그인되어 있어야 함.
- `config.json`의 `대상권역.목록`에서 `status: active`인 권역만 수집.

## 1. 네이버 부동산 — 후보 1차 수집 (Phase 1)
- new.land.naver.com 에서 대상 권역으로 이동.
- 사이트 필터 적용: 매매 / 가격 ~5억 / 면적(공급 20평↑=66㎡↑) / 세대수. (연식·용적률은 사이트 필터에 없으면 단지 상세에서 판정)
- 각 단지에서 수집: 단지명, 주소, 매매 최저가, 매물 개수, 매물 링크(단지 URL), 준공연도, 용적률, 대표 평형(공급/전용 ㎡·평), 세대수.
- → `data/raw/naver-<권역>-<날짜>.json`

## 2. 하드 필터 1차 판정
- 준공 ≥ 1996, 용적률 ≤ 250, 공급 ≥ 20평, 세대수 ≥ 50, 매매최저가 ≤ 5억 통과분만 다음 단계로.

## 3. 아실 — 실거래가 (Phase 2)
- asil.kr 에서 단지 검색 → 최근 실거래가(직전 거래가 + 최근 6개월 평균)와 거래일 수집.
- → `data/raw/asil-<단지>.json`

## 4. 네이버 지도 — 대중교통 시간 (Phase 2, 정밀 필터)
- map.naver.com 대중교통 길찾기: 출발=단지 주소, 도착=강남구청역.
- 소요시간(분)·환승 수 수집. **60분 초과면 여기서 탈락.**
- → `data/raw/transit-<단지>.json`

## 5. 부동산플래닛 — 노후도 (Phase 2)
- main.bdsplanet.com 에서 단지 조회 → 노후도 지표를 1줄 코멘트로 요약.

## 6. 호갱노노 — 실리뷰 (Phase 2)
- hogangnono.com 에서 단지 리뷰 → 대표 리뷰 1~2줄 코멘트로 요약(출처 표기).

## 7. (보조) 부동산지인·리치고·직방·S-MAP
- 필요 시 시세비교·재건축 가능성·조망 참고만. 필수 아님.

## 8. 통합 저장
- 단지별로 위 값을 합쳐 `data/apartments.json`의 `단지목록`에 upsert(단지 고유키=네이버 단지번호).
- `config.json`의 `수집이력` 갱신.

## 9. 점수·대시보드 (Phase 3~4)
- `apartments.json` 기준으로 선호 점수 계산 → `dashboard.html` 생성 → Artifact 게시.

## 자동 갱신 시 (Phase 5)
- 1번을 다시 돌려 신규/사라진 매물 감지 → 변동 단지만 3~6단계 재수집 → 점수 재계산 → 같은 URL 재배포. 변동분 하이라이트.
